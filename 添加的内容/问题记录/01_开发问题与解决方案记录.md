## 🚀 实战避坑 & 架构反思

### 01_Spring Security 架构错位与拦截器重构
- **问题描述**: 在 Phase 1 开始时，准备按照标准流程实现 `UserDetailsService`，但发现原厂项目并无此类。
- **定位过程**: 通过 `graphify` 扫描，发现原厂采用 `HandlerInterceptor` 实现轻量级 JWT 校验。
- **解决方案**: 废弃原本计划，改为复现 `UserAuthInterceptor`。
- **面试谈资**: 体现对**架构灵活性**的理解，不盲从框架。

### 02_洋葱架构中的路径冗余与包规范
- **问题描述**: 出现了 `infrastructure/infrastructure/common` 这种奇怪的路径。
- **解决方案**: 进行了手术级重构，确保 100% 对齐原厂 DDD 结构。
- **面试谈资**: 体现对 **DDD 规范** 和代码整洁度的极致追求。

### 03_Spring Boot 3.x 依赖缺失与 Servlet 升级
- **问题描述**: 无法导入 `jakarta.servlet` 类。
- **定位过程**: 确认 `pom.xml` 中仅有基础 starter，缺少 web starter。
- **解决方案**: 升级为 `spring-boot-starter-web`。
- **面试谈资**: 体现对 **Spring Boot 自动配置机制** 及新版本规范的快速适应能力。

### 04_Spring Boot Starter 与 Maven 传递依赖
- **场景**: 加入 Validation Starter 后自动获得了 `@NotBlank` 等注解。
- **深度原理**: 理解 Starter 作为一个“聚合包”，利用 Maven 的 **传递依赖 (Transitive Dependency)** 机制，将接口层（jakarta.validation-api）与实现层（hibernate-validator）一站式引入。

### 05_空 DTO (GetCaptchaRequest) 的设计哲学
- **场景**: 发现获取验证码的请求类为空类。
- **深度原理**: 
    - **统一契约**: 所有的 POST 请求统一使用 `@RequestBody` 对象，方便前端调用。
    - **开闭原则**: 为未来扩展（如验证码类型、长度）预留字段空间，而无需改动 Controller 方法签名。

### 06_高并发下工具类静态内部类与容器安全
- **场景**: 分析 `CaptchaUtils` 在高并发下的线程安全性。
- **核心辨析**: 
    - **JVM 层面**: 静态内部类由 JVM 保证线程安全且原子化加载。
    - **JMM 层面**: 必须关注对象属性的**可见性 (Visibility)**，推荐使用 `final` 关键字构建不可变对象，防止在多线程下读到未初始化的属性。
- **面试谈资**: 展示对 **Java 内存模型 (JMM)** 以及 **Happens-before 原则** 的底层理解。

### 07_MyBatis-Plus 配置层级陷阱与 Bean 注入失败
- **问题描述**: 启动项目报错 `UnsatisfiedDependencyException`，提示找不到 `UserRepository` 类型的 Bean。
- **定位过程**: 
    1. 确认 `UserRepository` 已继承 `BaseMapper`。
    2. 发现 `application.yaml` 中 `mybatis-plus:` 配置被错误地缩进在了 `spring:` 节点下（导致配置无效）。
    3. 发现部分 Repository 类（如 `UserSettingsRepository`）缺失了 `@Mapper` 注解。
- **深度原因**: 
    - **YAML 层级敏感性**: Spring Boot 无法识别非标准路径下的 `mybatis-plus` 配置，导致自动配置类（AutoConfiguration）跳过了 Mapper 扫描和实体类解析。
    - **Bean 注册机制**: 在没有显式 `@MapperScan` 的情况下，MyBatis 依赖 `@Mapper` 注解或配置文件的路径指引来生成动态代理类。
- **解决方案**: 
    - **修正层级**: 将 `mybatis-plus` 移至 YAML 根路径。
    - **加强扫描**: 在启动类添加 `@MapperScan("com.jin.agentx_learning")`。
    - **补全注解**: 为所有 Repository 接口显式添加 `@Mapper`。
- **面试谈资**: 体现对 **YAML 语法严谨性** 的关注，以及对 **Spring Boot 自动配置加载顺序**（尤其是第三方 Starter 如何读取配置）的深度理解。
