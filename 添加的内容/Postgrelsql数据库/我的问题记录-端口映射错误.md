# 我的问题记录 - PostgreSQL 端口映射错误

## 问题现象
- PostgreSQL 容器可以正常启动
- 可以用 `docker exec` 进入容器内部连接数据库
- 但 IDEA 或外部工具无法连接

## 错误配置
```bash
# ❌ 错误的端口映射
docker run --name postgres-container_AgentX_Learn \
  -e POSTGRES_USER=AgentX_Learn \
  -e POSTGRES_PASSWORD=AgentX_Learn \
  -e POSTGRES_DB=AgentX_Learn \
  -p 5433:5433 \          # 错误！容器内不监听 5433
  -v postgres_data:/var/lib/postgresql \
  -d postgres:latest
```

## 错误原因

### 1. 端口映射理解错误
端口映射格式：`主机端口:容器端口`

- **主机端口（Host Port）：** 你在 IDEA 中连接的端口（如 5433）
- **容器端口（Container Port）：** 容器内服务实际监听的端口

### 2. PostgreSQL 默认监听端口
PostgreSQL 容器内默认只监听 **5432** 端口，不会监听 5433！

从容器日志可以确认：
```
listening on IPv4 address "0.0.0.0", port 5432
```

### 3. 我的 docker ps 显示
```
PORTS
5432/tcp, 0.0.0.0:5433->5433/tcp
```
- `5432/tcp` - 镜像默认暴露的端口声明
- `0.0.0.0:5433->5433/tcp` - 我的错误映射（主机5433 -> 容器5433）

**问题：** 容器内 5433 端口没有服务在监听！

## 正确配置

```bash
# ✅ 正确的端口映射：主机 5433 -> 容器 5432
docker run --name postgres-container_AgentX_Learn \
  -e POSTGRES_USER=AgentX_Learn \
  -e POSTGRES_PASSWORD=AgentX_Learn \
  -e POSTGRES_DB=AgentX_Learn \
  -p 5433:5432 \          # 正确！容器内监听 5432
  -v postgres_data:/var/lib/postgresql \
  -d postgres:latest
```

## 解决步骤

```bash
# 1. 停止并删除错误的容器
docker stop postgres-container_AgentX_Learn
docker rm postgres-container_AgentX_Learn

# 2. 重新创建容器（用正确的端口映射）
docker run --name postgres-container_AgentX_Learn -e POSTGRES_USER=AgentX_Learn -e POSTGRES_PASSWORD=AgentX_Learn -e POSTGRES_DB=AgentX_Learn -p 5433:5432 -v postgres_data:/var/lib/postgresql -d postgres:latest

# 3. 确认端口映射正确
docker ps
# 应该看到：0.0.0.0:5433->5432/tcp
```

## IDEA 连接配置
- **Host:** `localhost`
- **Port:** `5433`（主机端口）
- **User:** `AgentX_Learn`
- **Password:** `AgentX_Learn`
- **Database:** `AgentX_Learn`
- **URL:** `jdbc:postgresql://localhost:5433/AgentX_Learn`

## 总结
- 端口映射格式：`主机端口:容器端口`
- PostgreSQL 容器内固定监听 **5432**（无法通过环境变量更改）
- 如果想在主机用其他端口（如 5433），映射格式必须是：`5433:5432`
