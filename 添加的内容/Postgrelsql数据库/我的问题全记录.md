# 我的 PostgreSQL Docker 配置问题全记录

## 问题总结

这次配置 PostgreSQL Docker 容器一共遇到了 **5 个主要问题**：

---

## 问题1：镜像下载后在文件夹里看不到文件

### 现象
- `docker pull postgres:latest` 下载成功
- 去 `C:\Users\jin\AppData\Local\Docker\wsl` 查看，看不到新增的镜像文件

### 原因
Docker Desktop 使用 WSL2，镜像存储在 WSL2 的虚拟磁盘（`.vhdx`）内部，不是直接以文件形式暴露在 Windows 文件系统中。

### 解决
- 用 `docker images` 命令查看已下载的镜像
- 不要在 Windows 文件夹里找镜像文件

---

## 问题2：docker pull latest 每次都重新下载

### 现象
- 之前下载过 postgres:latest
- 再次运行 `docker pull postgres:latest` 还是会下载内容

### 原因
`latest` 标签是**动态更新**的，官方会定期更新（安全更新、版本升级等）。如果本地的 latest 镜像和仓库的不一致，就会重新下载。

### 解决
- 这是正常现象，不是问题
- `docker images` 中 `CREATED` 列显示的是镜像在仓库构建的时间，不是下载时间

---

## 问题3：端口映射配置错误

### 现象
- 容器可以正常启动
- `docker exec` 可以进入容器连接数据库
- 但 IDEA 或外部工具连接失败

### 错误配置
```bash
# ❌ 错误
-p 5433:5433
```

### 原因
1. **端口映射格式理解错误**：`主机端口:容器端口`
2. **PostgreSQL 容器内默认只监听 5432**，不会监听 5433

### 正确配置
```bash
# ✅ 正确：主机 5433 -> 容器 5432
-p 5433:5432
```

### 验证
```bash
docker ps
# 应该看到：0.0.0.0:5433->5432/tcp
```

---

## 问题4：数据卷旧数据导致配置不生效

### 现象
- 重新创建容器，指定了 `POSTGRES_USER=AgentX_Learn`
- 但连接时报错：`Role "AgentX_Learn" does not exist`
- 容器日志显示：`Skipping initialization`

### 原因
PostgreSQL 初始化只在**数据卷为空**时执行一次。如果数据卷中已有数据库文件，后续创建容器时会跳过初始化，环境变量配置不生效。

### 日志证据
```
PostgreSQL Database directory appears to contain a database; Skipping initialization
FATAL:  password authentication failed for user "AgentX_Learn"
DETAIL:  Role "AgentX_Learn" does not exist.
```

### 解决

#### 方案一：删除旧数据卷（推荐）
```bash
docker stop postgres-container_AgentX_Learn
docker rm postgres-container_AgentX_Learn
docker volume rm postgres_data  # 关键！
docker run --name postgres-container_AgentX_Learn -e POSTGRES_USER=AgentX_Learn -e POSTGRES_PASSWORD=AgentX_Learn -e POSTGRES_DB=AgentX_Learn -p 5433:5432 -v postgres_data:/var/lib/postgresql -d postgres:latest
```

#### 方案二：使用新的命名卷
```bash
# 使用 postgres_data_new 代替 postgres_data
-v postgres_data_new:/var/lib/postgresql
```

---

## 问题5：多个 PostgreSQL 容器端口冲突疑问

### 疑问
- 容器1：`5432:5432`
- 容器2：`5433:5432`
- 会不会冲突？

### 答案
**不会冲突！**
- 主机端口分别是 5432 和 5433，不一样
- 容器内端口虽然都是 5432，但那是在各自容器内部，互相隔离

---

## 最终正确的完整命令

```bash
docker run --name postgres-container_AgentX_Learn \
  -e POSTGRES_USER=AgentX_Learn \
  -e POSTGRES_PASSWORD=AgentX_Learn \
  -e POSTGRES_DB=AgentX_Learn \
  -p 5433:5432 \              # 主机5433 -> 容器5432
  -v postgres_data:/var/lib/postgresql \  # Postgres 18+ 挂载上层目录
  -d postgres:latest
```

## IDEA 连接配置

- **Host:** `localhost`
- **Port:** `5433`
- **User:** `AgentX_Learn`
- **Password:** `AgentX_Learn`
- **Database:** `AgentX_Learn`
- **URL:** `jdbc:postgresql://localhost:5433/AgentX_Learn`

---

## 经验教训

1. **端口映射牢记格式：** `主机端口:容器端口`
2. **PostgreSQL 容器内固定监听 5432**，无法通过环境变量更改
3. **数据卷不为空时，初始化会跳过**，环境变量不生效
4. **Docker 镜像存储在 WSL2 虚拟磁盘中**，不要在 Windows 文件夹里找
5. **`latest` 标签是动态的**，会定期更新
6. **多个容器只要主机端口不同**，就不会冲突
