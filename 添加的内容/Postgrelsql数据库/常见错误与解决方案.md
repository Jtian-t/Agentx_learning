# PostgreSQL Docker 常见错误与解决方案

## 问题1: 数据卷挂载旧数据导致配置不生效

### 问题描述
- 使用相同的命名卷（如 `postgres_data`）重新创建容器
- 新容器的环境变量（POSTGRES_USER、POSTGRES_PASSWORD、POSTGRES_DB）不生效
- PostgreSQL 仍使用旧数据中的配置

### 原因
PostgreSQL 初始化只在**数据卷为空**时执行一次。如果数据卷中已有数据库文件，后续创建容器时会跳过初始化，环境变量配置不生效。

### 解决方案

#### 方案一：删除旧数据卷重新创建（推荐，会丢失旧数据）
```bash
# 1. 停止并删除容器
docker stop postgres-container_AgentX_Learn
docker rm postgres-container_AgentX_Learn

# 2. 删除旧数据卷
docker volume rm postgres_data

# 3. 重新创建容器（会自动创建新的数据卷）
docker run --name postgres-container_AgentX_Learn \
  -e POSTGRES_USER=AgentX_Learn \
  -e POSTGRES_PASSWORD=AgentX_Learn \
  -e POSTGRES_DB=AgentX_Learn \
  -p 5433:5432 \
  -v postgres_data:/var/lib/postgresql \
  -d postgres:latest
```

#### 方案二：使用新的命名卷
```bash
# 使用不同的卷名，如 postgres_data_new
docker run --name postgres-container_AgentX_Learn \
  -e POSTGRES_USER=AgentX_Learn \
  -e POSTGRES_PASSWORD=AgentX_Learn \
  -e POSTGRES_DB=AgentX_Learn \
  -p 5433:5432 \
  -v postgres_data_new:/var/lib/postgresql \
  -d postgres:latest
```

## 问题2: 端口映射配置错误

### 问题描述
- IDEA 无法连接到 PostgreSQL
- 容器内 PostgreSQL 正常运行，但外部连不上

### 原因
端口映射格式理解错误：
- 格式：`主机端口:容器端口`
- **容器内 PostgreSQL 默认监听 5432 端口**，不是 5433！

### 错误示例
```bash
# 错误：容器内 PostgreSQL 不监听 5433
-p 5433:5433
```

### 正确配置
```bash
# 正确：主机端口 5433 -> 容器端口 5432
-p 5433:5432
```

### 完整命令
```bash
docker run --name postgres-container_AgentX_Learn \
  -e POSTGRES_USER=AgentX_Learn \
  -e POSTGRES_PASSWORD=AgentX_Learn \
  -e POSTGRES_DB=AgentX_Learn \
  -p 5433:5432 \
  -v postgres_data:/var/lib/postgresql \
  -d postgres:latest
```

### 验证端口映射
```bash
# 查看容器端口映射
docker ps
# 应该看到类似：0.0.0.0:5433->5432/tcp
```

## 问题3: 端口冲突导致容器启动失败

### 问题描述
- 容器创建成功，但启动失败
- 提示端口已被占用

### 原因
Docker 执行 `run` 命令分两步：
1. 先创建容器（分配名字、配置、卷挂载）
2. 再尝试启动容器 + 绑定端口

如果第二步端口绑定失败 → 容器已经创建好了，只是启动失败。

### 解决方案
```bash
# 1. 查看失败的容器（包括停止的）
docker ps -a

# 2. 删除创建失败的容器
docker rm postgres-container_AgentX_Learn

# 3. 检查端口占用情况
netstat -ano | findstr :5433

# 4. 改用其他端口，或停止占用端口的程序
# 修改为 -p 5434:5432
```

## 问题4: Postgres 18+ 数据卷挂载警告

### 问题描述
- 容器启动时出现数据目录警告
- 提示挂载目录不正确

### 原因
PostgreSQL 18 之后存储目录规则变化，直接挂载 `/var/lib/postgresql/data` 会有警告。

### 解决方案
改为挂载上层目录：
```bash
# 旧方式（Postgres 18 前）
-v postgres_data:/var/lib/postgresql/data

# 新方式（Postgres 18+）
-v postgres_data:/var/lib/postgresql
```

## 排查问题通用步骤

### 1. 检查容器状态
```bash
# 查看运行中的容器
docker ps

# 查看所有容器（包括停止的）
docker ps -a
```

### 2. 查看容器日志
```bash
docker logs postgres-container_AgentX_Learn
```

### 3. 进入容器本地测试
```bash
docker exec -it postgres-container_AgentX_Learn psql -U AgentX_Learn -d AgentX_Learn
```

### 4. 检查端口映射
```bash
docker ps
# 确认 PORTS 列显示正确的映射
```
