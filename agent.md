# AgentX 复现项目 - 核心协作契约 (v2.12)

## 🎯 项目愿景 (The Goal)
基于 `lucky-aeon/AgentX` 的最终形态进行 100% 像素级复现，并进行架构深挖与优化。

---

## 📜 十一大神圣执行协议 (Sacred Protocols)

### 1. 严禁擅自简化与总结协议 (No-Summary Constraint)
- **准则**: 忠于原文。
- **动作**: 对于用户未提出修改要求的任何内容（包括但不限于协议细节、TODO 任务描述、技术参数），Agent **严禁**进行自主的合并、缩写、概括或总结。
- **目的**: 确保所有规则和步骤保持其最详尽的初始状态，严防指令在迭代中“信息丢失”或“逻辑走样”。

### 2. 上下文锚定与内容保护协议 (Contextual Anchoring)
- **准则**: 笔记即现场。
- **动作**: 严禁将用户在特定任务（如 2.1 节）下的笔记、链接、感悟移动到其他位置或进行全局总结。
- **要求**: Agent 的补充建议必须以 `> **By Gemini:**` 开头，且必须紧贴在对应的任务条目之后，确保“问题-解决方案-感悟”三位一体，严禁切断上下文。

### 3. 经验沉淀与分包协议 (Experience Anchoring & Partitioning)
- **准则**: 让每一个 Bug 都有价值。
- **动作要求**:
  - **实时性**: 每当遇到阻碍或重大的架构修正，立即更新。
  - **深度性**: 不仅记录“怎么做”，更记录“为什么”以及“底层原理”。如果描述的是某一个类的问题，**要写出该类的完整路径**。
  - **面试化**: 总结每个问题的“关键词”，作为未来面试的谈资。
- **容量控制**: 当单个记录文件过大时，必须开启新文档进行分层存储。文档路径在 `添加的内容/问题记录` 下。

### 4. 架构优化咨询协议 (Architectural Optimization Consulting)
- **触发条件**: 当用户对某个类或功能提出“优化设计”时执行。
- **产出动作**: 在 `添加的内容/改进方案/` 下创建 `数字递增_类名.md` 文档。
- **内容结构**: 必须包含：[涉及类]、[现状分析]、[优化方案]、[核心优势]。
- **准则**: 只给文档，不改代码。

### 5. 原子化任务分解协议 (Atomic Task Specification)
- **准则**: 计划即代码，描述即逻辑。
- **动作要求**: 任何新增的类任务，必须严格遵循以下结构描述（严禁简化）：
  - **[类名]**: 完整的 `org.xhy...` 路径。
  - **[职责]**: 一句话描述该类在 DDD 架构中的核心功能。
  - **[前置依赖]**: 显式列出其 import 链中所有属于本项目的新增类。
  - **[关键注解/配置]**: 标注如 `@Data`, `@RestController`, `@NotBlank`, `@Email` 等核心细节。

### 6. 依赖透明协议 (Dependency Transparency)
- **准则**: 凡事预则立。
- **动作**: 在实现 Controller 或复杂 Service 实现前，必须在计划中“穷举”其所有依赖类（DTO/Utils），确保护航开发链路无死角。

### 7. 图谱先行协议 (Graph-First Discovery)
- **准则**: 优先图谱定位，雷达式搜索。
- **动作**: 架构查询必须优先执行 `graphify query`，提高定位精确度。

### 8. 参考优先协议 (Reference-First Protocol)
- **准则**: 像素级还原原厂实现。
- **动作**: 强制基于图谱定位到 `reference/AgentX` 下的原厂实现。

### 9. 数字标识与还原协议 (Identity & Restoration)
- **准则**: 独立编号（从 `01_` 开始）；包名还原。

### 10. 自动化同步协议
- **准则**: 每完成一步，必须同步进度至 GitHub 及数据库 `project_progress` 表。

### 11. 严禁静默删除协议 (Zero-Silent-Deletion) - *NEW*
- **准则**: 每一个文件都是开发资产。
- **动作**: 严禁在未明确告知用户并获得确认的情况下，删除、移动或大幅度重构任何已有文件。
- **要求**: 即使发现路径错误或冗余，Agent 必须先在对话中说明：“发现 X 文件路径错误，建议清理并迁移至 Y”，待用户确认后再执行删除操作。
